## 1. QA 방법 (Methodology)

### 1.1 자동 브라우저 루프 — Claude Code × Playwright MCP
- `.mcp.json`에 Playwright(Chrome) MCP를 붙여 **Claude Code가 실제 크롬을 직접 운전**.
- 루프: **탐색 → 콘솔/네트워크 관찰 → 결함·미구현 식별 → 코드 보완 → 동일 시나리오 재검증 → 다음 페이지**.
- 매 변경마다 `typecheck`/`lint` 회귀 확인 + 브라우저로 동작 재검증.

### 1.2 OAuth 우회 — 세션 시드
- 세션 전략이 `database` → `scripts/qa-login.ts`로 **DB에 세션을 심고 쿠키만 주입**해 OAuth 없이 권한 단계별 로그인(normal/approved/admin).
- 권한 승인·소유자 관리 등 인증 플로우를 사람 개입 없이 검증.

### 1.3 페이지 감사 — 병렬 에이전트
- 미구현/미완 기능 발굴 시 **Explore 에이전트 3개를 병렬**로 페이지 그룹 감사 → 결과를 종합해 백로그화.

### 1.4 수동 시나리오 — Claude for Chrome
- 실제 브라우저를 보는 에이전트가 따라 실행할 **시나리오 문서**(전제→단계→기대→결과)를 작성.
- 자동(쿠키주입)이 못 하는 실 OAuth/시각 회귀를 사람·에이전트가 보완.

### 1.5 트리아지 원칙
- 보고된 결함을 **코드+브라우저로 검증**해 ❌반박 / ⚠️강등 / 🔧설계결정 / ✅수정 으로 분류 → 과잉수정·오인 방지.
