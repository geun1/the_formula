# 사용자 시나리오 테스트 (E2E 여정)

페르소나가 **목표**를 가지고 여러 페이지를 넘나드는 실제 사용 흐름을 검증한다.
[CHROME_SCENARIOS.md](CHROME_SCENARIOS.md)가 기능 단위 점검이라면, 이 문서는 "한 사람이 처음부터 끝까지 써보는" 통합 여정이다.

형식: **페르소나·목표 → 전제 → 여정(단계) → 성공 기준 → 연관 케이스**
클로드인크롬/사람 모두 그대로 따라 실행 가능. 로그인 필요 여정은 테스터가 먼저 로그인해 둔다.

> 계정 상태 표기: `비로그인` / `로그인(일반)` / `로그인(승인)` / `로그인(관리자)` — 자세한 정의는 [CHROME_SCENARIOS.md](CHROME_SCENARIOS.md) §0.

---

## US-1 · 처음 방문한 사람이 둘러보고 가입을 결심한다
**페르소나**: AI 업무 활용에 관심 있는 첫 방문자 · **목표**: 사이트가 뭔지 파악하고 가입을 결정
**전제**: 비로그인

**여정**
1. `/` 접속 → 히어로 문구와 "인기 글 TOP 5" 를 읽는다.
2. 관심 카테고리(예: 개발)를 사이드바에서 눌러 글을 좁혀 본다.
3. 마음에 드는 글 제목을 눌러 아티클 상세를 읽는다 → 원문 링크/댓글 분위기를 살핀다.
4. 다른 사람들이 남긴 공식이 궁금해 헤더 "아카이브"로 이동, 카드 몇 개를 본다.
5. 멤버가 궁금해 "포뮬러(멤버)"로 이동, 직무별로 둘러본다.
6. 하단 "가입하고 시작하기"(또는 헤더 가입하기) 클릭 → 가입 화면 도달.

**성공 기준**: 비로그인으로 탐색이 막힘 없이 이어지고, 모든 페이지 콘솔 에러 0, 마지막에 가입 동선이 명확히 보인다.
**연관**: HOME-1~4, ART-1, ARC-1, MEM-1, APL-1

## US-2 · 새 회원이 가입 직후 자리를 잡는다(온보딩 → 프로필)
**페르소나**: 막 가입한 신규 회원 · **목표**: 온보딩을 마치고 내 프로필을 갖춘다
**전제**: 로그인(일반), 온보딩 미완료 상태가 이상적

**여정**
1. 로그인 직후 `/onboarding` 진입(또는 `/account` → "온보딩" 링크).
2. 직무 선택 + 관심 키워드 2~3개 선택 → "시작하기".
3. 홈으로 이동됨을 확인.
4. `/account` 이동 → 소개·소속·외부 링크(GitHub/블로그) 입력 후 "변경사항 저장".
5. `/profile/me` 이동 → 방금 입력한 정보·외부 링크가 내 프로필에 노출되는지 확인.

**성공 기준**: 온보딩 완료 후 추천/피드가 보이고, 프로필 편집이 저장되어 내 공개 프로필에 반영된다.
**연관**: ON-1, PRO-2, PRO-1, PRO-3

## US-3 · 학습자가 좋은 공식을 발견하고 내 것으로 만든다
**페르소나**: AX를 배우려는 실무자 · **목표**: 쓸 만한 공식을 찾아 저장하고 따라해 본다
**전제**: 로그인(일반)

**여정**
1. 헤더 검색창에 키워드(예: "자동화") 입력 → 검색 결과 확인.
2. (결과가 없으면) `/archive`로 가서 검색창에 "리팩토링" 입력 → 카드 필터.
3. 공식 카드 제목 클릭 → 상세에서 문제~결과 흐름을 읽는다.
4. "공식 저장" 클릭(하트/카운트 토글 확인).
5. "따라하기" 클릭 → 내 복제본 상세로 이동 확인.
6. `/profile/me`에서 "북마크한 공식"/"내 공식"에 반영됐는지 확인.

**성공 기준**: 검색→상세→저장→복제가 끊김 없이 이어지고, 내 프로필 통계에 반영된다.
**연관**: SR-1, ARC-1~2, FM-1~2, PRO-1

## US-4 · 회원이 토론에 참여한다(댓글 → 대댓글 → 정리)
**페르소나**: 의견을 나누고 싶은 회원 · **목표**: 글에 댓글·답글을 달고, 잘못 단 건 지운다
**전제**: 로그인(일반)

**여정**
1. 아티클 상세 진입 → 댓글 입력 → "등록"(최상단 노출 확인).
2. 방금 단 내 댓글의 "답글" → 답글 작성 → "답글 등록"(중첩 확인).
3. 다른 사람 댓글에는 삭제 버튼이 **없음**을 확인(내 것에만 있음).
4. 내가 단 답글의 "삭제" → 확인창 확인 → 답글 사라지고 카운트 감소.

**성공 기준**: 작성/답글/삭제가 즉시 반영되고, 권한(본인만 삭제)이 UI로 지켜진다.
**연관**: ART-2~4, SEC-2

## US-5 · 창작자가 자기 공식을 발행하고 관리한다
**페르소나**: 자신의 AI 업무 노하우를 공유하려는 회원 · **목표**: 공식을 쓰고, 필요 없어지면 지운다
**전제**: 로그인(일반)

**여정**
1. `/archive/new` 진입 → 제목 입력.
2. "📋 가이드 형식"으로 문제/가설/도구/과정/결과 작성 → "공식 올리기".
3. 발행된 내 공식 상세로 이동됨을 확인.
4. 상세 상단 액션 바에 **삭제** 버튼이 보이는지 확인(내 글이라).
5. "삭제" → 확인창 확인 → `/archive` 목록으로 이동, 글이 사라짐.

**성공 기준**: 비개발자도 가이드 폼으로 공식을 발행할 수 있고, 본인 글 삭제가 동작한다.
**연관**: ARC-3, FM-3, SEC-2

## US-6 · 큐레이터 지망생이 아티클 추가 권한을 얻는다(요청 → 승인 → 추가)
**페르소나**: 좋은 외부 아티클을 큐레이션하고 싶은 회원 + 운영자 · **목표**: 권한 승인 흐름 전체
**전제**: 회원=로그인(일반), 운영자=로그인(관리자) — 계정 2개 사용

**여정**
1. [회원] `/article/new` 진입 → "권한이 필요해요" 안내 확인 → 메모 입력 → "권한 요청하기"(검토중 전환).
2. [운영자] `/article/new` 진입 → 하단 "권한 요청 관리"에서 그 회원 요청 확인 → **승인**.
3. [회원] `/article/new` 새로고침 → 이제 URL 추가 폼이 보임(승인됨).
4. (선택, 관리자/승인) 기사 URL 입력 → "아티클 추가하기" → 생성 확인. *AI 키 없으면 실패 메시지 — 환경 의존.*
5. (거절 흐름) 운영자가 다른 요청을 **거절** → 그 회원 화면에 "거절됐어요, 다시 요청 가능" → 재요청 시 검토중 복귀.

**성공 기준**: 요청→승인으로 권한이 실제 부여되고, 거절→재요청 루프가 막히지 않는다.
**연관**: ART-6~8, P-1~P-4, SEC-2

## US-7 · 호스트가 스터디를 열고 지원자를 관리한다
**페르소나**: 모임을 만들고 운영하려는 회원 · **목표**: 모임 개설 → 지원 받기 → 승인/마감
**전제**: 호스트=로그인(일반), 지원자=다른 로그인 계정

**여정**
1. [호스트] `/activities/new` → 구분/인원/제목/소개 입력 → "등록하기"(새 모임 상세로 이동).
2. [지원자] 그 모임 상세 진입 → 지원 메시지 입력 → "지원하기"("검토 중" 표시).
3. [지원자] 빈 메시지로도 시도 → 검증 에러로 막히는지 확인.
4. [호스트] 모임 상세 "내가 만든 모임이에요" 패널에서 지원자 확인 → **수락**(상태 변경).
5. [호스트] 모집을 닫으려 **진행중**으로 상태 전환("마감" 배지 확인).
6. [호스트] 테스트 종료 후 **삭제**로 모임 정리(확인창 → 목록 복귀).

**성공 기준**: 개설→지원→승인→상태전환→삭제까지 호스트 운영 루프가 끊기지 않는다(지원자가 검토중에 방치되지 않음).
**연관**: ACT-2~7, Y-1~2

## US-8 · 참가자가 사람을 찾아 연결한다(검색 → 팔로우 → 채팅)
**페르소나**: 동료를 찾는 회원 · **목표**: 관심 멤버를 팔로우하고 1:1 대화를 시작
**전제**: 로그인(일반)

**여정**
1. `/members`에서 검색창에 이름/직무로 멤버 검색.
2. 멤버 카드의 "팔로우" 클릭 → "팔로잉"으로 전환.
3. 같은 카드의 "채팅"(또는 프로필의 채팅 요청) → 1:1 대화방 진입.
4. 메시지 입력 → "보내기" → 스레드에 내 메시지 노출.
5. `/chat` 받은함으로 가서 방금 대화가 미리보기로 보이는지 확인.
6. (해제) 그 멤버를 다시 "팔로잉" 눌러 언팔로우 → "팔로우"로 복귀.

**성공 기준**: 검색→팔로우→채팅→받은함이 하나의 흐름으로 자연스럽게 이어진다.
**연관**: MEM-2~4, CHAT-1~2

## US-9 · 비로그인 사용자가 벽에 부딪힌다(권한·경계 확인)
**페르소나**: 로그인하지 않은 사용자 · **목표**: 쓰기 액션 시 적절히 가입/로그인으로 유도되는지
**전제**: 비로그인

**여정**
1. 아티클 상세 → 댓글을 달려 하면 입력창 대신 "로그인이 필요해요" 안내.
2. 공식 상세 → "저장/따라하기" 시 로그인 유도로 연결.
3. `/article/new` 직접 접근 → 추가 폼이 아니라 로그인/권한 안내.
4. 존재하지 않는 경로 접근 → 친절한 404(홈/아카이브 바로가기).

**성공 기준**: 모든 쓰기/보호 액션이 우회 없이 막히고, 길 잃은 사용자에게 회복 동선이 제공된다.
**연관**: SEC-1~3

## US-10 · 운영자가 하루를 점검한다(운영 스모크)
**페르소나**: 사이트 운영자(송근일) · **목표**: 들어온 요청·콘텐츠를 빠르게 점검
**전제**: 로그인(관리자)

**여정**
1. `/article/new` → 대기 중 권한 요청들을 확인하고 승인/거절 처리.
2. `/` 피드 → 정렬을 인기순으로 바꿔 최근 인기 글 점검.
3. `/archive` → 새로 올라온 공식 점검.
4. `/activities` → 모집 중 모임 현황 점검.
5. `/members` → 신규 멤버 유입 확인.

**성공 기준**: 운영자 관점의 점검 동선이 한 바퀴 매끄럽게 돌아가고, 관리자 전용 UI(권한 요청 관리)가 정상 노출된다.
**연관**: ART-7, HOME-2, ARC-1, ACT-1, MEM-1

## 실행 가이드

- **우선순위**: US-1(첫인상) · US-7(모임 운영) · US-6(권한) · US-8(네트워킹)이 핵심 가치 흐름 → 먼저 돌린다.
- **회귀(짧게)**: US-1 → US-4 → US-8 한 바퀴면 핵심 사용자 경험 스모크가 된다.
- **계정 준비**: US-6·US-7은 계정 2개(요청자/운영자, 호스트/지원자) 필요. 미리 로그인 세션을 준비.
- **기록**: 각 여정 PASS/FAIL + 막힌 단계 번호. FAIL은 [BUGS.md](BUGS.md), 흐름상 빠진 기능은 [UNIMPLEMENTED.md](UNIMPLEMENTED.md)에 연결.
