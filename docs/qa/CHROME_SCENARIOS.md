# 클로드인크롬 테스트 시나리오 + 실행 결과

브라우저를 직접 보고 조작하는 에이전트(Claude for Chrome)가 그대로 따라 실행할 수 있는 시나리오 모음 + 최신 실행 결과.
각 시나리오는 **전제 → 단계 → 기대 → 결과** 형식. 화면에서 눈으로 확인 가능한 것만 검증한다.

---

## 0. 사용 안내 (먼저 읽기)

- **기본 URL**: `http://localhost:3002` (운영 테스트 시 운영 도메인으로 치환)
- **로그인 방식**: 소셜 로그인(현재 네이버 활성). 에이전트는 쿠키 주입을 못 하므로,
  **로그인이 필요한 시나리오는 테스터가 먼저 로그인**해 두고 시작한다.
- **계정 상태별 전제** (시나리오에 표기):
  - `비로그인` — 로그아웃 상태
  - `로그인(일반)` — 로그인했지만 아티클 추가 권한 없음
  - `로그인(승인)` — 운영자가 아티클/AI 작성 권한을 승인한 계정
  - `로그인(관리자)` — `ADMIN_USER_IDS`에 등록된 운영자 계정(송근일)
- **모든 페이지 공통 체크(C-ALL)** — 각 시나리오에서 항상 같이 본다:
  1. 페이지가 깨지지 않고 정상 렌더되는가
  2. 개발자도구 콘솔에 에러(빨강)가 없는가
  3. 클릭한 링크/버튼이 의도한 동작을 하는가(아무 일도 안 일어나는 곳 없는가)
- **결과 기록**: 각 시나리오의 **결과** 줄에 표기. FAIL은 [BUGS.md](BUGS.md)에 등록.

## 1. 홈 `/`

### HOME-1 · 홈 렌더 · 전제: 비로그인
1. `/` 접속.
2. 헤더에 로고, 네비(아티클·아카이브·포뮬러·모임), 검색창, **로그인/가입하기** 버튼 확인.
3. 본문에 "인기 글 TOP 5", 카테고리 사이드바, 글 목록 노출 확인.
- **기대**: 위 요소 모두 노출, 콘솔 에러 0.
- **결과**: ✅ PASS — 전 요소 노출, 콘솔 0.

### HOME-2 · 정렬 변경
1. 글 목록 위 정렬 셀렉트를 **인기순**으로 변경.
2. URL이 `/?sort=popular` 로 바뀌고 목록 순서가 바뀌는지 확인.
3. **저장순**으로도 변경해 본다.
- **기대**: 선택값에 따라 목록 재정렬, URL 반영.
- **결과**: ✅ PASS — `?sort=` URL 반영·재정렬.

### HOME-3 · 카테고리 필터
1. 사이드바에서 "개발" 클릭 → URL `/?category=dev`, 목록이 해당 카테고리로 필터되는지 확인.
2. "전체"로 복귀.
- **기대**: 카테고리별 필터 동작.
- **결과**: ✅ PASS — 필터·하이라이트·복귀 정상.

### HOME-4 · 네비게이션 탭
1. 헤더 "모임" 클릭 → `/activities` 이동.
2. "아카이브" 클릭 → `/archive` 이동.
3. "포뮬러" 클릭 → `/members` 이동.
- **기대**: 각 목적지로 정상 이동, 404 없음.
- **결과**: ✅ PASS — 전 탭 이동, 404 없음.

### HOME-5 · 로그인 진입
1. 헤더 "로그인" 클릭 → 로그인 화면(소셜 버튼) 노출 확인.
- **기대**: 네이버 로그인 버튼 노출(설정된 provider만).
- **결과**: ✅ PASS — "네이버로 계속하기" 노출.

## 2. 아티클 `/article/[id]`, `/article/new`

### ART-1 · 상세 렌더 · 전제: 비로그인
1. 홈에서 아무 글 카드 제목 클릭 → 상세 진입.
2. 제목, 요약, 본문/카드뉴스, 원문 링크, 댓글 섹션 노출 확인.
- **기대**: 정상 렌더, 콘솔 에러 0.
- **결과**: ✅ PASS — 전 요소 노출, 콘솔 0.

### ART-2 · 댓글 작성 · 전제: 로그인(일반)
1. 아티클 상세 하단 댓글 입력창에 텍스트 입력.
2. 입력 전에는 "등록" 버튼이 비활성, 입력 후 활성되는지 확인.
3. "등록" 클릭.
- **기대**: 댓글 수 +1, 내 댓글이 목록 최상단에 즉시 노출, 입력창 비워짐.
- **결과**: ✅ PASS — 등록 활성화, 3→4, 최상단 노출, 입력창 초기화.

### ART-3 · 대댓글(답글) · 전제: 로그인(일반)
1. 내 댓글의 "답글" 클릭 → 답글 입력창 펼쳐짐.
2. 답글 입력 후 "답글 등록" 클릭.
- **기대**: 부모 댓글 아래 들여쓰기된 위치에 답글 노출.
- **결과**: ✅ PASS — 부모 아래 중첩.

### ART-4 · 본인 댓글 삭제 · 전제: 로그인(일반)
1. 내가 쓴 댓글에 **삭제** 버튼이 보이는지 확인(남의 댓글엔 없어야 함).
2. "삭제" 클릭 → 같은 자리에 **"취소 / 삭제"** 2단계 확인이 펼쳐짐(네이티브 팝업 아님).
3. "삭제"(빨강)를 한 번 더 클릭.
- **기대**: 해당 댓글(달린 답글 포함) 제거, 댓글 수 감소. "취소" 누르면 삭제 안 됨.
- **결과**: ✅ PASS — 본인 댓글만 삭제 버튼, 대댓글 포함 삭제. (2단계 확인은 UX-2 수정 반영)

### ART-5 · 댓글 페이지네이션 · 전제: 댓글 10개 초과 글
1. 댓글이 많은 글에서 하단 "이전 댓글 더 보기" 클릭.
- **기대**: 추가 댓글 로드, 중복/누락 없음.
- **결과**: ⏭ 미실행(클로드인크롬) · ※자동화 PASS(12개 시드→더보기).

### ART-6 · 추가 권한 — 권한 없음 · 전제: 로그인(일반, 권한없음)
1. `/article/new` 접속.
2. "아티클 추가 권한이 필요해요" 안내 + 요청 메모 입력 + "권한 요청하기" 노출 확인(추가 폼은 없어야 함).
3. 메모 입력 후 "권한 요청하기" 클릭.
- **기대**: "⏳ 권한 요청이 검토 중이에요(pending)"로 전환.
- **결과**: ✅ PASS — 요청 UI만 노출, 요청→검토중 전환.

### ART-7 · 추가 권한 — 관리자 승인 · 전제: 로그인(관리자)
1. `/article/new` 접속 → "큐레이션 · 관리자" + URL 추가 폼 + 하단 "권한 요청 관리(N)" 노출 확인.
2. 대기 요청의 **승인** 클릭.
- **기대**: 요청 목록에서 사라지고 카운트 감소(해당 유저가 승인됨).
- **결과**: ⏭ 미실행(관리자 계정 필요) · ※자동화 PASS(P-2/P-3).

### ART-8 · URL로 아티클 추가 · 전제: 로그인(관리자/승인)
1. `/article/new` 의 "기사 URL"에 실제 기사 URL 입력 → "아티클 추가하기" 클릭.
2. 처리(크롤→AI 요약) 대기.
- **기대**: 새 아티클 생성·피드 노출(AI 큐레이터 작성). *AI 키 미설정 시 실패 메시지 — 환경 의존.*
- **결과**: ⏭ 미실행(관리자/승인 + AI 키 ENV-1 필요).

### ARC-3 · 공식 작성(가이드 형식) · 전제: 로그인(일반)

### ARC-5 · AI 초안 생성 · 전제: 로그인(승인)

- **기대**: 초안 생성 후 편집 가능. *AI 키 미설정 시 "AI 초안 생성에 실패했어요…" 메시지(원본 에러 노출 없어야 함).*
