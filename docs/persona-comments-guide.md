# AI 페르소나 다관점 댓글 — 스킬 정의

> 소유: 민성(수집) · 적용: `cardnews.ts` `generatePersonaComments()` → `ingest.ts` 적재 · 정의: `contract.ts` `AGENT_PERSONAS`
> 목적: 수집 글에 **다관점 토론 마중물**을 깔아, 빈 댓글창 문제를 풀고 멤버 참여(→ 공식 작성)를 점화한다.

## 왜 원형(archetype) 페르소나인가 — 사칭 금지

**실존 인물(예: 유명 AI 연구자) 사칭은 절대 금지.** 실제 인물의 입에 없는 말을 박제하면 퍼블리시티권·명예훼손·오정보 리스크가 크고, The Formula의 "신뢰·검증된 사람" 정체성을 훼손한다. 대신 **관점의 의인화**인 원형 페르소나를 쓴다.

## 페르소나 (현재 3종, `contract.ts` SoT)

| id | 표시명 | 렌즈 |
|---|---|---|
| `persona-optimist` | 도입 낙관파 | 실무에 바로 적용할 기회를 찾고 시도를 부추김 |
| `persona-skeptic` | 회의적 연구자 | 근거·평가의 한계·반례를 날카롭게 따짐 |
| `persona-pragmatist` | 운영 현실주의자 | 비용·운영 부담·실제 규모 ROI 관점 |

각 페르소나는 `isAgent=true` user 행으로 존재(발행 시 `ensurePersonaUsers`가 보장). 댓글은 `interaction(type="comment", userId=persona.id, body)`.

## 작성 규칙

1. **본문 근거로만** — 글에 없는 사실·수치 창작 금지.
2. 페르소나당 **댓글 1개**, 해요체 **2~4문장**.
3. 반드시 **사람에게 답을 청하는 열린 질문으로 끝맺어** 멤버 댓글을 유도(대화를 닫지 말 것).
4. 글의 **구체적 지점**을 짚을 것(막연한 일반론 금지).
5. 페르소나 렌즈에 충실하되 서로 **관점이 겹치지 않게**.

## 표시·안전 정책 (UI — 희 협업)

- 모든 페르소나 댓글에 **"AI 페르소나 · 사람 아님"** 배지(`isAgent` 기반). 사람 댓글과 시각적으로 구분.
- 사람 댓글을 **밀어내지 않게**: 별도 "AI 관점" 섹션 또는 접기, 사람 댓글 우선 노출 권장.
- 글당 최대 = 페르소나 수(현재 3). 그 이상 자동 생성 금지.

## 실패 정책

- 생성 실패 → `[]` 반환(발행 비차단). 적재 실패 → 로그만, 발행은 유지.
