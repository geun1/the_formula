### A-2. 레이아웃/컴포넌트 차이
1. **헤더**: 로고 + 아티클/아카이브/포뮬러/모임 + **검색 input 바**(공식·포뮬러·모임 검색) + 로그인/가입하기. (우리: 검색 아이콘만 → input 바로 교체)
2. **홈 히어로**: **다크 네이비(#071330) 풀폭 밴드** + eyebrow(요일/날짜) + 큰 흰 제목("지금 가장 해결하고 싶은 업무 고민은?") + 서브 + **큰 검색 박스**. (우리: 라이트 블루 카드 → 다크 히어로+검색으로 교체)
3. **인기 글 TOP 5**: 카드가 **다크 히어로 하단에 겹쳐 올라옴**(레이어드). 카드: 번호+카테고리·툴칩+제목+작성자+**👁조회·💬댓글·♥좋아요**.
4. **홈 본문 레이아웃**: **2컬럼** = 관심 카테고리(좌) + **3열 카드 그리드**(우측 위젯 없음). (우리: 3컬럼+우측 위젯 → 2컬럼으로. 모집중모임/추천포뮬러는 모임 페이지로 이동)
5. **카드**: 커버 그라데이션+카테고리칩 / 작성자·날짜 / 제목 / 설명 / 카테고리+툴칩 + **👁조회 💬댓글 ♥좋아요 + 🔗공유(링크복사) 아이콘**. (우리 카드에 **조회수·공유 추가**)
6. **정렬**: **드롭다운(최신순 ▾)** (우리: 탭 → 드롭다운)
7. **공식/아티클 상세**: **2컬럼** = 본문(좌) + **우측 사이드바**[작성자 카드 + 프로필보기 + **팔로우 + 채팅** + "작성자의 다른 공식" 위젯]. 액션(**공식 저장·따라하기**)은 **상단**. 프롬프트는 **다크 코드블록 + 복사** 버튼. 상세엔 커버이미지 없이 텍스트 집중.
8. **포뮬러(멤버)**: "전체 디렉토리 N명" + 카테고리(아이콘) 사이드바 + 3열 카드[아바타+이름+직무 + **멤버저장(하트)** + bio + 관심칩 + **공식 N·저장 N** + **팔로우 + 채팅**].
9. **푸터**: **다크 4컬럼**(브랜드+태그라인 / 커뮤니티 / 안내 / 소셜) + "AX Community" "Beta v0.9" 뱃지. (우리: 단순 푸터 → 4컬럼 다크)
10. **로그인**: 카카오/네이버 소셜(우리와 동일) — 유지.

## B. 기능 차이 (분류 + 적용)

### B-1. 신규 기능 — 백엔드 필요
- **★ 채팅 / 1:1 DM**: 멤버·작성자에게 "채팅" 버튼 → 1:1 대화. (우리: "커피챗" 링크 자리만) → conversation/message 테이블 + 보내기/목록/대화 UI(+폴링 갱신) + 받은함(inbox).
- **멤버 저장(bookmark member)**: 멤버 카드 하트 → 멤버 저장. → member_bookmark 테이블(또는 follow 재활용) + UI.

### B-2. 신규/강화 — 가벼움(주로 FE + 기존 데이터)
- **조회수(view) 표시·집계**: 상세 진입 시 view interaction +1, 카드/TOP5에 👁 표시. (우리: view interaction 있으나 미표시 → 표시·집계)
- **공유(링크 복사)**: 카드/상세에 🔗 공유 → 클립보드 복사 토스트. (신규, client)
- **헤더/히어로 검색 input**: 입력 → /search?q= 이동. (우리: 아이콘만 → input)
- **작성자의 다른 공식 위젯**: 상세 우측, 작성자의 다른 글 목록. (쿼리 추가)

### B-3. 우리가 가진 것(레퍼런스가 약함) — 유지/공존
- **신뢰등급(매너온도)**: 레퍼런스는 공식수·저장수만 노출. → 우리는 **공식수·저장수(토스풍) 전면 + 등급 뱃지 보조**로 공존(제거하지 않음).
- 아티클↔아카이브 연결, 수집 파이프라인 API, 모임/온보딩: 유지.
